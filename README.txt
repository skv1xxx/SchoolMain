Название проекта
Веб-сайт для хостинга видео

Описание проекта
Django-приложение для загрузки, хранения и просмотра видеоконтента с системами пользователей, категорий и поиска. Проект включает полную аутентификацию пользователей, загрузку медиафайлов и административную панель.

Основные функции
Пользовательская система:
-Регистрация с email и паролем
-Авторизация
-Личный кабинет
-Загрузка аватаров

Управление видео:
-Загрузка видеофайлов
-Добавление превью (миниатюр)
-Привязка к категориям
-Поиск по названию и описанию

Категории:
-Создание категорий
-Просмотр видео по категориям
-Безопасное удаление категорий (без потери видео)

Административная панель:
-Управление пользователями
-Управление видео
-Управление категориями

Технологический стек
Backend: Django 5.2
База данных: SQLite3
Фронтенд: HTML, CSS (шаблоны Django)
Аутентификация: Кастомная модель пользователя на основе AbstractBaseUser
Медиафайлы: Локальное хранение с валидацией расширений

Модели базы данных
1. BDuser (Пользователь)
nickname - уникальный никнейм
name, surname - имя и фамилия
email - уникальный email (используется для входа)
photo - аватар пользователя
date_joined - дата регистрации
is_active, is_staff, is_superuser - флаги доступа

2. BDvid (Видео)
title - название видео
description - описание
publish_time - время публикации
video_file - файл видео
thumbnail - превью видео
category - связь с категорией (ForeignKey)

3. BDkat (Категория)
name - название категории (уникальное)
thumbnail - изображение категории

Установка и запуск
1. Предварительные требования
Python 3.8+
Django 5.2

2. Клонирование и настройка

# Клонирование проекта (если используется git)
# git clone <repository-url>
# Создание виртуального окружения
python -m venv venv

# Установка зависимостей
pip install django

# Если есть requirements.txt:
# pip install -r requirements.txt

3. Настройка базы данных

# Применение миграций
python manage.py makemigrations
python manage.py migrate

4. Создание суперпользователя

python manage.py createsuperuser

*Следуйте инструкциям для создания администратора.

5. Запуск сервера

python manage.py runserver

Сервер будет доступен по адресу: http://127.0.0.1:8000/

Доступные URL-адреса
/ - главная страница (все видео)

/admin/ - административная панель

/reg/ - регистрация

/ent/ - вход в систему

/logout/ - выход из системы

/acc/ - личный кабинет (требует входа)

/upload/ - загрузка видео (требует входа)

/kat/ - список категорий

/category/<id>/ - видео конкретной категории

/video/<id>/ - просмотр конкретного видео

/search/ - поиск видео

/design/ - страница дизайна

/about_us/ - страница "О нас"

/repass/ - восстановление пароля

Настройки проекта
Основные настройки в mysite/settings.py:

DEBUG = True - режим отладки (для продакшена установить False)

ALLOWED_HOSTS - разрешенные хосты

AUTH_USER_MODEL = 'vhost.BDuser' - кастомная модель пользователя

MEDIA_URL и MEDIA_ROOT - настройки для медиафайлов

LANGUAGE_CODE = 'ru-ru' - русский язык интерфейса

TIME_ZONE = 'Europe/Moscow' - московское время

Особенности реализации
Безопасность:
-Хеширование паролей через make_password
-Валидация загружаемых файлов по расширениям
-CSRF защита
-Авторизация для защищенных страниц через @login_required

Администрирование:
-Кастомные действия в админ-панели
-Безопасное удаление категорий (видео не удаляются)
-Русскоязычные названия моделей

Обработка медиа:
-Раздельное хранение: аватары, видео, превью
-Автоматическая генерация URL для миниатюр
-Валидация форматов: jpg, jpeg, png для изображений

Разработка:
Добавление нового функционала
Создайте модель в models.py

Создайте миграции: python manage.py makemigrations

Примените миграции: python manage.py migrate

Создайте форму в forms.py (если нужно)

Создайте представление в views.py

Добавьте URL-маршрут в vhost/urls.py

Создайте HTML-шаблон в templates/

Тестирование

# Запуск тестов
python manage.py test vhost
